## Архитектура и взаимодействие фич

Приложение следует принципам чистой архитектуры и разделено на независимые фичи. Взаимодействие между ними осуществляется через общие блоки управления состоянием (`BLoC`) и DI-контейнер (`GetIt`).

## Основные фичи:
1.  **All Fruits (`all_fruits`)**: Основная фича, отвечающая за загрузку, отображение и управление списком всех фруктов.
2.  **Favourites (`favourites`)**: Отображение списка избранных фруктов.
3.  **Sorting (`sorting`)**: Предоставление функционала для сортировки и фильтрации списка фруктов.
4.  **Recipes (`recipes`)**: Создание рецептов на основе избранных фруктов.
